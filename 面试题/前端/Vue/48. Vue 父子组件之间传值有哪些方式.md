在 Vue 中，父子组件之间传值是常见的操作，以下为你详细介绍几种常见的传值方式：

### 父组件向子组件传值
#### 1. `props`
- **原理**：`props` 是 Vue 中用于父组件向子组件传递数据的主要方式。父组件在调用子组件时，通过属性绑定的方式将数据传递给子组件，子组件通过 `props` 选项接收这些数据。
- **使用步骤**
    - 子组件在 `props` 选项中定义要接收的属性，可以指定属性的类型、默认值等。
    - 父组件在使用子组件时，通过 `v - bind`（简写为 `:`）将数据绑定到子组件的属性上。

#### 2. 非 `props` 的特性
- **原理**：当父组件传递给子组件的属性没有在子组件的 `props` 选项中定义时，这些属性会作为非 `props` 的特性自动绑定到子组件的根元素上。
- **适用场景**：当子组件需要接收一些通用的 HTML 属性（如 `class`、`style` 等）时，使用非 `props` 的特性可以简化代码。

### 子组件向父组件传值
#### 1. 自定义事件
- **原理**：子组件通过 `$emit` 方法触发自定义事件，并传递数据，父组件在使用子组件时，通过 `v - on`（简写为 `@`）监听这些事件，并在事件处理函数中接收子组件传递的数据。
- **使用步骤**
    - 子组件在需要传递数据时，使用 `this.$emit('eventName', data)` 触发自定义事件并传递数据。
    - 父组件在使用子组件时，使用 `@eventName="handleEvent"` 监听事件，并在 `handleEvent` 方法中接收数据。

#### 2. 事件总线（Event Bus）
- **原理**：事件总线是一个全局的事件发射器，它可以在不同组件之间传递事件和数据。创建一个事件总线实例，子组件通过事件总线触发事件并传递数据，父组件通过事件总线监听事件并接收数据。
- **使用步骤**
    - 创建一个事件总线实例，通常是一个新的 Vue 实例。
    - 子组件在需要传递数据时，使用事件总线的 `$emit` 方法触发事件并传递数据。
    - 父组件在需要接收数据时，使用事件总线的 `$on` 方法监听事件并接收数据。

### 父子组件之间的双向数据绑定
#### `v - model`
- **原理**：`v - model` 是 Vue 提供的一种语法糖，用于实现父子组件之间的双向数据绑定。它结合了 `props` 和自定义事件，父组件通过 `v - model` 绑定数据到子组件，子组件通过触发 `input` 事件更新父组件的数据。
- **使用步骤**
    - 子组件在 `props` 中定义 `value` 属性，并在需要更新数据时触发 `input` 事件。
    - 父组件在使用子组件时，使用 `v - model` 绑定数据。
